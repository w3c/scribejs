issue_directives

  • issue_directives(config: Configuration, directive: string, issue_references: string): IssueInformation
    • Defined in lib/issues.ts:37

    Handling the scribejs, issue X,Y,Z type directives. The method returns a set of strings to be added to the final (markdown) minutes. Two information items are returned

    1. A standard markdown line is added listing the the issue URL-s.
    2. If this works with Jekyll, a comment is returned with the list of URL-s in the form of <!-- issue URL_X,URL_Y,URL_Z -->

    If the original directive just includes a - instead of the issue numbers a special HTML/markdown comment returned of the form <!-- issue - -->.

    While the comments are invisible when displayed as part of the minutes, they can be used by postprocessing tools that gathers a portion of the discussions that is relevant to a particular issue or set of issues.

    Parameters

    • config: Configuration

      the general scribejs configuration object

    • directive: string

      either 'pr' or 'issue'

    • issue_references: string

      comma separated list of issue numbers, possibly of the form repo#number

    Returns IssueInformation

    the markdown lines to be added to the final minutes

  • titles(config: Configuration, content: string): Promise<IssueReference>
    • Defined in lib/issues.ts:137

    Handling a (sub)topic line: find, if present, a @issue XX,YY,ZZ line and process it as in issue_directives leaving the rest of content as the real (sub)topic line.

    Parameters

    • config: Configuration

      the general scribejs configuration object

    • content: string

      the content of the full title line

    Returns Promise<IssueReference>

    • object with title_text set to the title line content (issue number if missing) and issue_reference to the list of issues to display (empty string if missing)
